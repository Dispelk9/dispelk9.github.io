Datapipeline: Is this CI/CD?

08.July.2025
The request to update the database daily made me how to create a pipeline.
In this request I have to create a SQL dump from a active repository which changed daily
This repository contains a lots of information which are essential for the INFOSEC but they can not access them directly therefore we need to upload them to a GUI which they as users can access daily using their own credential.
For this pipeline, I will use the Gitlab CICD, which hold that repository, adjust the gitlab_ci to create an task:
- Using docker to import the repository using gitlab runner, using import script to import all yaml files in this repo and convert them to proper table.
- after the database is created, extract that database as the dump file and saved them as artifacts.
- create another task to upload that to S3.

When the file is upload to S3, there are two options to restore them to Aurora/RDS
- Using AWS EC2 and using AWS CLI there to do the restore, because the RDS and other components are in the private VPC, therefore we cannot just use AWS CLI locally, which make unnecessary provisioning a EC2 just for that.
- Using AwS Lambda with a self-created layer (Using Dockerfile) for import using python
With the new Lambda, it can get an trigger from S3 whenever S3 get the new artifacts, it will run this Python script to restore the new data to RDS
Then last thing to do is schedule the pipeline x2 daily on Gitlab CICD
Now you have a complete Gitlab Pipeline including #s3#gitlab_ci#aws_lambda#aws_aurora_rds#python#Docker

#Hetzner#Github actions = Profit?

22.Feb.2024
Hetzner and GitHub Actions are both popular tools in the DevOps ecosystem, and they complement each other well in various aspects of the development and deployment pipeline. That's why I choose them to begin my Devops Journey. It is clear to me that a lots will choose AWS (because of their popularity) and Gitlab( it is free, why not?). First AWS free tier is not really free if you have no idea what you are doing with it, so it is not really something I suggest for a beginner, and you are really have to take care of the billing. Hetzner based in Germany, and it is not too bad to support the local cloud :D.
Gitlab is really popular in corporation but in order to take care of self hosted runner then I thinks it is not really good for beginner. In case you have some self hosted Gitlab cloud server somewhere to runner 24/7 then it is fine.
Anyway, the first CI/CD of mine are used with these combo. Using Terraform to provision a server on Hetzner that runs Analytical Tools, which used Flask, for my wife. I am pretty happy with it as it is a first step of mine on this devops/cloud journey.

#Difference#Helpdesk#Sysadmin

17.Oct.2023
It would be nice if everyone can understand what is the difference between these jobs.
Helpdesk typically deal with problems that involve the desktop/laptop and pheripherals (mouse, headphone, and maybe even PRINTER (the devil if you know)), basically the work involves more with the users.
Sysadmin keeps the whole IT infrastructure going, scripting, maintaining, updating the system so that it won't get down whenever happens (hacking,attacking). Logging and Data collecting are also part of your jobs incase your Product Owner/Technical IT Manager demands it.
So now you understand what is the difference between two of them, then what will you choose?
